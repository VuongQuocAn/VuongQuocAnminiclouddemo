Docker Container

Từ sinh viên chưa biết gì về container đến xây dựng thành công hệ thống MyMiniCloud với 9 server chỉ trong một dự án.

Khi bắt đầu học môn Điện toán Đám mây, tôi hoàn toàn không hình dung được rằng Docker sẽ trở thành một phần quan trọng đến vậy trong hành trình của mình.

Hình 1: Kiến trúc Container so với Virtual Machine

1. Docker là gì?

Hình 2: Tổng quan về Docker

Định nghĩa cơ bản

Theo nguồn Wikipedia: "Docker là một dự án mã nguồn mở giúp tự động triển khai các ứng dụng Linux và Windows vào trong các container ảo hóa."

Theo nhiều diễn đàn công nghệ: "Docker là một open platform cung cấp cho người sử dụng những công cụ và service để có thể đóng gói và chạy chương trình của mình trên các môi trường khác nhau một cách nhanh nhất."

Nói một cách đơn giản, chúng ta sẽ thiết lập cho ứng dụng của mình các môi trường cần thiết với Docker. Khi ai đó muốn chạy ứng dụng đó, họ chỉ cần chạy Docker mà không cần phải cài đặt thủ công hàng tá cấu hình phức tạp. Bạn không cần phải biết về Laravel, Ruby on Rails hay Java vẫn có thể chạy được dự án một cách trơn tru!

Các thành phần chính của Docker

  • Docker Engine: Dùng để tạo ra Docker image và chạy Docker container.
  • Docker Hub: Dịch vụ lưu trữ trực tuyến giúp chứa và chia sẻ các Docker image.

Một số khái niệm quan trọng

Docker Machine

Công cụ tạo ra các Docker engine trên máy chủ hoặc trực tiếp trên hạ tầng cloud.

Docker Compose

Chạy multi-container docker applications bằng cách định nghĩa cấu hình qua tệp docker-compose.yml.

Giá trị cốt lõi Docker mang lại

  • Đóng gói đồng nhất: Đóng gói mọi ứng dụng (webapp, backend, MySQL...) thành các containers chạy ổn định ở “hầu hết” môi trường (Linux, Mac, Windows).
  • API linh hoạt: Docker Containers cung cấp API cho phép quản trị từ bên ngoài, dễ dàng quản lý và chỉnh sửa.
  • Tương thích cao: Hầu hết các ứng dụng Linux đều có thể chạy mượt mà với Docker Containers.
  • Tốc độ vượt trội: Khởi động nhanh cấp số nhân so với các VMs truyền thống (Hypervisor). Đây chính là ưu điểm ăn tiền nhất!

2. Sự khác biệt giữa Docker và Hypervisors

Hình 3: So sánh kiến trúc Docker Container vs Virtual Machines (Hypervisor)

Dự án yêu cầu phải xây dựng đủ 9 server chạy song song trong cùng một mạng cloud-net. Từ Web Server (Nginx), Application Backend (Flask), Database (MariaDB), Keycloak Authentication, MinIO Object Storage, cho đến Prometheus + Grafana Monitoring và Reverse Proxy. Nếu chạy bằng Hypervisors (VMs), cấu hình yêu cầu sẽ rất nặng nề và cồng kềnh. Với Docker, tất cả đều nhỏ gọn, cách ly độc lập và chia sẻ chung OS kernel của máy host.

3. Sự khác biệt giữa Docker Image và Docker Container

Docker Images

Là một template chỉ cho phép đọc (read-only). Mọi image có thể chứa hệ điều hành Ubuntu và cài sẵn một web app chẳng hạn. Images được dùng làm khuôn mẫu để tạo ra các Docker container. Docker cho phép build và cập nhật các image tự định nghĩa, hoặc download public image từ người khác (Docker Hub).

Docker Containers

Là một phiên bản ứng dụng đang chạy thật (runtime environment). Nó có cơ chế nét giống với thư mục hệ thống. Một container giữ mọi thứ cần thiết để ứng dụng hoạt động không lỗi. Container được khởi tạo từ image tương ứng, và có các vòng đời: run, started, stopped, moved, và deleted.

“Docker không chỉ giúp triển khai nhanh hơn, mà còn thay đổi hoàn toàn cách chúng ta nghĩ về hạ tầng và môi trường chạy ứng dụng.”
